# 04. Plano de Testes

## Objetivo
Planejar a execução dos testes do produto com foco em risco e rastreabilidade.

## Entradas
- user stories aprovadas
- critérios de aceite definidos
- ambiente disponível
- massa de dados configurada

## Saídas
- evidências de execução
- bugs registrados
- parecer de qualidade por entrega

## Ciclos
- smoke por merge
- regressão antes do marco final
- reteste após correção de defeitos
